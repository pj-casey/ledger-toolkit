Quick Start

File to diagnosis in 30 seconds.

1

Load the log. Drag the customer's log file onto the tool, or click Browse. Accepts .json .txt .log

2

Read the Diagnosis block at the top of the sidebar. green = clean session. yellow = warnings. red = critical issues.

3

Click the relevant section in the sidebar: Issues, Accounts, Timeline, or Advanced.

4

For richer data, switch to Customer View and load the customer's app.json.

The tool analyzes the log automatically. The Diagnosis block tells you what it found — you don't need to search for problems.

Investigation Playbook

Start with the scenario that matches your customer's complaint.

"My balance is wrong / I'm missing funds"

Start here: Switch to Customer View → load the customer's app.json for exact balances.

Look for: Compare displayed balance with expected. Check if tokens are showing. Look for pending operations.

Next: If no app.json available, check Accounts in Diagnostic mode for sync status. Search Timeline for sync to see if synchronization completed.

If app.json shows a different balance than Ledger Wallet, the local cache may be stale. Ask the customer to clear cache and resync.

"My transaction failed / is stuck"

Start here: Click Issues in the sidebar. Look for error cards mentioning rejected, failed, or insufficient funds.

Look for: Severity borders on error cards — red is critical. Click "Affected account" to see which account. Click "Jump to timeline" for the exact event sequence.

Next: Check Advanced → Network for failed API calls (4xx/5xx status codes). If the error mentions device rejection, check Advanced → APDU.

APDU rejection codes 6985 or 6982 mean the device itself rejected the transaction — the user didn't approve on-screen or conditions weren't met.

"My device won't connect / isn't recognized"

Start here: Check Issues for hardware/USB errors. Look for "Manager connection" failures.

Look for: Device model and firmware version in the Overview. If the model says "—", the device never identified itself during this session.

Next: Check Advanced → APDU for device communication. If there's zero APDU data, the device never established a connection at all.

No APDU data = no device communication during this session. Check USB cable, port, or Bluetooth before investigating software.

"The app crashes / won't open / is slow"

Start here: Read the Diagnosis block. The severity and issue count tell you how bad this session was.

Look for: Critical errors (red). Repeated patterns — the same error appearing many times suggests a loop. Check Timeline for the event sequence leading up to the problem.

Next: Check Advanced → Network for server errors (5xx). Timeouts or connectivity failures point to a server-side issue, not the app.

Green diagnosis but customer reports crashes? The log may not capture the crash. Ask them to reproduce and export logs immediately after.

I need to verify what the customer sees

Start here: Switch to Customer View using the top bar toggle.

Look for: Portfolio overview with all accounts. Click any account for full detail — address, tokens, transaction history.

Next: Load the customer's app.json via the upload banner. Adds real balances, tx history, and account names.

General investigation (no specific complaint)

Start here: Read the Diagnosis block for the overall picture.

Recommended flow:

Issues → what went wrong
Accounts → what exists
Timeline → what happened when
Advanced → deeper data if needed

Problems first, context second, sequence third.

The Interface

Two modes, switchable from the top bar toggle.

Diagnostic Mode — "Technical investigation"

Sidebar on the left for navigation. Main area on the right for detail. The sidebar is always visible — you never lose your place.

🔍
Diagnosis Block
Severity dot, summary sentence, Copy dropdown. Always at the top. Click to return to Overview.
⚠️
Issues
Errors and warnings. Severity borders, affected account links, jump-to-timeline. Expandable details on each card.
👛
Accounts
Crypto accounts with chain color dots, operation counts, explorer links. Error badges show which accounts have problems.
📋
Timeline
Every event, chronological. Searchable, filterable. Error/warning rows have colored left borders.
🌐
Network
HTTP requests to Ledger and third-party APIs. Status codes, error summary.
📡
APDU
Device communication commands. Status codes, rejection summaries. Only when APDU data exists.
{ }
Raw JSON
Interactive tree viewer. Search keys and values, expand/collapse levels, copy any JSON path.
📊
Overview
Default view. Stat cards, device info, environment, activity badges, issues preview.

Customer View — "See the customer's wallet"

Read-only wallet simulation. Account list on the left, portfolio or account detail in the center, device info on the right. Load app.json via the upload banner for exact balances and transaction history.

Red "mismatch" warning after loading app.json? You loaded the wrong customer's file. Click ✕ to remove and upload the correct one.

Copy & Export

Three formats

Copy Summary Quick ticket note. Device info, top accounts, top errors. Paste into your support ticket.

Copy Full Complete data dump. All accounts, all errors. For escalation to Tier 2 or engineering.

Copy Customer Summary Account-focused. Balances and transaction counts. For describing what the wallet looks like. Available in Customer View.

Find these in the Copy dropdown in the sidebar, or the buttons at the top of the Overview.

Tips & Troubleshooting

The file won't load

Accepts .json, .txt, and .log files. Must be a Ledger Wallet log export. Large files show a loading indicator — give it a moment.

I see 0 accounts

Some logs don't include account data. Load the customer's app.json in Customer View for full account information.

All errors say "Unexpected error"

The error isn't in the tool's knowledge base. Expand the error card's "Details" section for the raw data, or search in Timeline or Raw JSON.

Customer View is empty

Make sure you've switched modes using the top bar toggle. If no accounts appear, load the customer's app.json.

App.json says "Ledger Sync encryption"

The file is encrypted by Ledger Sync. The tool extracts what it can (settings, paired devices, token preferences) but can't read balances or transaction history.

Diagnosis is green but customer has problems

The tool only sees what's in the log. If the issue happened outside this session, it won't be captured. Ask the customer to reproduce and export logs immediately after.

I loaded the wrong app.json

Click ✕ on the green banner to remove it, then upload the correct file.

The xpub scanner is slow

It queries the blockchain for each extended public key. Normal for UTXO accounts (Bitcoin, Litecoin, Dogecoin) with many transactions.
